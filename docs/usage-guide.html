The 60-second pitch

You give it a theme. Claude scans every transcript, returns the best moments with timestamps anchored to the start of a sentence, and lets you assemble a clip list to export for your editor. Built for shipping themed compilations off @Arab's longform catalog.

The four tabs

01 Library

Browse every video. Use AI search to rank by relevance to a topic (e.g. "eating crazy food" → rat poison, hallucination honey, jungle cuisine). Each card has a 0–100 score badge. Click any video for the player + full transcript.

02 Workspace

The build view. Type a theme → Find moments → Claude returns clip cards grouped by video. Click a thumbnail to preview in the persistent right-side player. Click + Add to assemble.

03 Titles

Brainstorm viral compilation titles. Claude proposes 8 options with alt titles + source video suggestions. Save the ones you like, or Build to drop into the Workspace.

04 Saved

All your in-progress compilations. Click any clip thumbnail to preview inline. Resume to keep working, Export CSV for your editor, Delete when done.

Tip — Title style. The AI is trained to avoid "I Spent X Days With Y" (Arab's own vlog hooks) and listicle phrasings. It produces multi-story-implying titles like "Times Brazil Got Out of Control" or "Why You Don't Mess With Favelas" — viral without screaming compilation.

Two ways to start a compilation

From a Title

  1. Go to Titles
  2. Type a seed theme or leave blank
  3. Click Generate ideas
  4. Pick one → Build
  5. You're dropped into the Workspace with the title pre-loaded and moments already searching

Directly in Workspace

  1. Go to Workspace
  2. Type a theme like "moments locals turned on him"
  3. Click Find moments
  4. Add clips you like — first + Add creates a new untitled compilation automatically
  5. Rename it inline anytime

How clip extraction actually works

Each video's transcript is split into ~45-second chunks, but each chunk keeps the underlying line-level YouTube auto-caption timing. When you search, the AI:

  • Stage 1 — ranks all 106 video titles + descriptions for relevance to your theme
  • Stage 2 — scans the top 8 picked videos' transcripts in parallel
  • Returns 1–5 minute moments with start times that match a line boundary — clips never start mid-word

The persistent inline player

One iframe for the whole app. Clicking different clip thumbnails seeks via postMessage instead of reloading. You can switch between 8 clips in 8 seconds without YouTube re-loading once.

State persists across tabs

Your theme, generated ideas, search results, and active compilation are saved to localStorage. Navigate to Library, come back, everything's still there. Same after refresh.

AI Library search

Plain text search only matches title/description substrings. AI search (Sparkles icon) sends your query + the whole 106-video index to Claude with a relevance scoring rubric.

  • 90–100 = dominant subject
  • 70–89 = strong match
  • 40–69 = touched on
  • <25 filtered out

Adding new @Arab uploads

On the Library page, hit + Add video. Paste any YouTube URL or 11-char ID. The server:

  1. Fetches metadata via YouTube API
  2. Validates channel = @Arab
  3. Shells out to yt-dlp for auto-captions
  4. Parses VTT, dedupes rolling-caption duplicates, chunks at ~45s
  5. Appends to catalog.json + chunks.json

Available in the next AI search instantly.

CSV export format

Each row in the export is one clip: #, video title, full URL, URL with ?t=Xs jump, in-timecode (HH:MM:SS), out-timecode, duration_sec, note (the quoted line).

Saved Titles vs Saved Compilations

  • Titles = an idea + suggested videos, no clips assembled yet. Bookmarks for later.
  • Saved = an actual compilation with picked clips, timestamps, ready to export.

Keyboard shortcuts

  • Enter in the workspace theme input — Find moments
  • Enter in the titles theme input — Generate ideas
  • Esc in any modal — Cancel
  • Enter in delete modal — Confirm
